frame_stacker.js

A helper library for managing iframe-based modal dialogs/overlays (hereinafter known as "frames"). Frames are managed in a stack-like manner. Only the top-most frame is interactive.

Installation

Get the latest release:
curl -L -O https://unpkg.com/frame_stacker/dist/frame_stacker.js

Add the script to your page and you're good to go.

Documentation

The demo on this page is a good starting point, but the source code serves as the official documentation. We recommend cloning and reading your local copy:
git clone --depth 1 https://github.com/quadrant-newmedia/frame_stacker.js.git

Demo

This page serves as the demo/test page. Most of the buttons on this page use in-line click handlers, and display the value of that handler right in the button.

On this page, and in every iframe we push, we've set:
window.fs = window.parent.frame_stacker;
On this page, window===window.parent, so this just creates a shorter alias for frame_stacker. In the iframes we push, window.parent refers to this window, so fs becomes a reference to this window's frame_stacker. That way, when code inside an iframe executes fs.push(...), it will actually push a frame onto this window. This our recommended mode of operation.

Accessibility / Usability

We aim to implement all of the accessibility features outlined on Mozilla's article about the ARIA "dialog" role.

  • When pushing an iframe, that iframe is focused within the parent page.
  • When an iframe page loads, the first element with the autofocus attribute will be given focus within the iframe.
  • Keyboard focus is trapped in the top-most frame.
  • Keyboard focus is restored after resolving a frame (whichever element had focus before the frame was pushed will be refocused when it is closed).
  • Click events outside the top-most frame are blocked.
  • The iframe automatically gets role="dialog" and aria-lable attributes. The aria-label attribute's value comes from the <title> of the iframe's document.

External Clicks/easy_exit

frame_stacker adds a backdrop element behind the top-most iframe, which stops you from activating buttons, etc.

By default, clicking outside the top-most frame does nothing. If you set exit_on_external_click on one of the plugins you pass to frame_stacker.push(), then clicking outside of it will cause it close. We recommend using the frame_stacker.easy_exit plugin, which sets exit_on_external_click to true, and also closes the frame if the user hits the escape key.

auto_layer

auto_layer combines a few different plugins, which can also use individually. It keeps the iframe centered in the viewport (though it isn't dead-center vertically - we think it looks better when it's closer to the top), and automatically sizes the iframe based on its content. The auto sizing feature should work for any combination of margin, padding, and box-sizing on the iframe element or on the body element of its document. We do not currently support margin or padding on the html element (documentElement) of the iframe (I've never seen anyone do that).

Plugins

Rather than taking a single configuration object, frame_stacker.push() accepts one or more configuration objects, which we refer to as "plugins".

You've already seen two plugins in use - frame_stacker.auto_layer

Refer to combine_plugins.js for a list of available plugin options.

Multiple plugins can be combined into a single, reusable plugin via frame_stacker.combine_plugins(plugin1, plugin2, ...). Several of the standard plugins we export are created this way.
